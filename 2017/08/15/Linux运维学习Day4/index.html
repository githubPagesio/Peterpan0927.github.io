文件管理

cp命令：

cp SRC DEST(src是单个文件)

1.如果目标不存在，新建DEST，并将SRC中的内容填充进去

2.如果目标存在：

如果是文件，就会将原内容覆盖，最好使用选项-i（交互式，避免误操作）

如果是目录，就新建一个和原文件同名的文件，然后将内容填充进去

cp SRC… DEST(src是多个文件)

这个时候DEST必须存在且为目录，其他情况均会出错

cp SRC DEST(src是目录)

这个时候要使用选项-r

1.如果目标不存在，创建指定的目录，复制SRC目录中所有的文件至DEST中

2.如果目标存在，if是文件，直接报错，if是目录，直接将SRC目录中的文件复制过去

其中--preserve选项太长了，可以写为-p,-v选项用来显示过程，不必多说

mv命令

mv SRC DEST

1.如果目标文件不存在，就创建一个同名文件，然后将src中的数据复制过去，并删除src的源文件

2.如果目标文件存在：

如果是目标是文件，就是覆盖

如果是目录，就新建一个同名文件，然后复制数据，删除源文件

mv SRC… DEST

同cp命令的这种情况，如果移动的是目录，不需要额外的添加选项

tips:如果mv到同一个目录下，可以视作重命名

mv命令常用选项也就是-i、-f

rm命令

常用选项：

-r、-f

用于删除文件，基本的使用很简单，但是要注意的是使用递归删除风险很大，使用需谨慎。

bash 的基础特性(2)

1.命令别名

通过alias命令实现；

alias:显示当前shell进程所有可用的命令别名

alias NAME=’VALUE’：定义别名NAME,其相当于执行命令VALUE

unalias:取消命令的别名

注意：在命令行中定义的别名，仅对当前shell进程有效，如果要永久有效，要定义在配置文件中去：

当前用户有效：~/.bashrc

对所有用户有效：/etc/bashrc

在修改了配置文件之后，创建新的进程才会读取配置文件，如果想要当前进程也更改，可以使用命令:

#如让别名生效
source .bashrc

如果别名同原命令的名称，则要执行原命令，可使用”\COMMAND”

glob(globbing)

bash中用于实现文件名”通配”

通配符：*，？，[]

1.*：任意长度的任意字符

a*b:aab、a123b、ab

2.?：任意的单个字符

rm -rf ?.py

3.[]：匹配指定范围内的任意的单个字符

[0-9]、[a-z]不区分大小写、[A-Z]大写字母

4.[^]:匹配指定范围外的任意单个字符

ls a[^0-9a-z]b

bash的I/O重定向和管道

打开的文件都有一个fd:file descriptor（文件描述符）

标准输入：keyboard，0

标准输出：monitor(显示器)，1

标准错误输出： monitor，2

I/O重定向也就是改变标准位置：

1.输出重定向：

‘>’ : command > NEW_POS 覆盖重定向，目标文件中原有内容会被消除

‘>>’ : command >> NEW_POS 追加重定向，新内容会追加到目标文件尾部

‘2>’ ： command 2> NEW_POS 将错误输出流覆盖重定向

‘2>>’: command 2>> NEW_POS 将错误输出流追加重定向

#禁止将内容覆盖输出至已有文件
set -C
#允许将内容覆盖输出至已有文件

如果要强制覆盖则使用 >|

如果要使正确、错误输出到不同位置:

#如果是错误输出流则只会执行前面的，但是两个文件都会创建
cat /etc/ssh/ssh_config > /tmp/testfile 2> /tmp/testfile2 
#但是上述的写法太过麻烦，可以合并标准和错误输出为同一个数据流进行覆盖重定向，追加重定向同理
echo $PATH &> /tmp/path.out
#还有一种写法看起来更清晰一点
echo $SHELL > /tmp/path.out 2>&1

Linux的安全上下文

运行中的程序：进程（process）以进程发起者的身份去执行

进程所能够访问的所有资源的权限取决于进程发起者的身份

/etc/passwd:

​name : passwd : UID: GID: GESOS : directory : shell

/etc/group:

​group_name : password : GID : user_list

组名：密码：GID：以当前组为附加组的用户列表（分隔符为逗号）

/etc/shadow

用户名：加密了的密码 ：最近一次更改密码的日期 ：密码的最小使用期限 ： 最大密码使用期限 ：密码警告时间段 ：密码禁用期 ：账号的过期日期

加密机制：

加密：明文—>密文

解密：密文—>明文

单向加密：提取数据指纹

md5:message digest , 128bits

sha1:secure hash algorithm（安全的哈希算法）,160bits

sha224:224bits

sha256…sha512等·都是一样的道理

如果在linux中有两个用户的登录密码是相同的，所以为了避免密码被猜出，密码在被加密的时候会被加上一段杂质，然后这样再加密生成的密码就完全看不出来了。

用户管理

切换用户或以其他用户身份执行命令

切换用户的方式：

su UserName:非登录式切换，即不会读取目标用户的配置文件

su - UserName：登陆式切换，会读取目标用户的配置文件，完全切换

su - UserName -c ‘COMMAND’ 换个身份执行命令

用户/组属性修改

usermod [options] login

这个命令的选项和useradd差不多，只不过意思是重新设置相关的属性，比如

usermod -g 2000 peterpan #表示重新设置用户peterpan的基本组
usermod -G mygrp peterpan #新附加组，原来的附加组将会被覆盖，若要保留原有，则要同时使用-a（append）选项
usermod -d /homepath  #新的家目录，原有的目录中的文件不会同时移动到新的家目录，如果要移动，要同时加上-m选项
usermod -l login_name #新的名字
usermod -L user_name  #lock指定用户
usermod -U user_name  #解锁指定用户

groupmod [options] group

groupmod -n pan peterpan  #新名字
groupmod -g 200 peterpan  #修改GID

passwd username

因为passwd命令是修改/etc/shadow文件，所以可以修改在这个文件中参数，如修改密码的时间等

gpasswd

gpasswd -a user mygrp #将user添加至指定用户组

gpasswd -A user1,user2 mygrp  #设置有管理权限的用户列表

权限管理

文件的权限管理主要针对三类对象进行定义：

owner、group、other

每个文件针对每类访问者都定义了三种权限：r,w,x

文件：

​ r:可使用文件查看类工具获取其内容

​ w:可以修改其内容

​ x:可以把此文件提请内核启动为一个进程

目录：

​ r:可以获取此目录下的文件列表

​ w:可以在在此目录中创建文件，也可以删除此目录中的文件

​ x:可以使用ls -l命令查看此目录中的文件列表，也可以cd进入此目录

修改文件权限

#修改权限的几种写法
chmod -R 775 /tmp/test #-R表示递归修改权限
chmod u(+/-)x /tmp/test    #修改一类用户的单个或某些位权限  
chmod u=rwx /tmp/test  #修改一类用户的所有权限 
chmod --reference=test.py test2.py. #以一个文件的权限为参照修改另一个文件的权限

修改文件的属主和属组

#其中选项的用法可以参考chmod
chown tom fstab  #将文件的属主改为tom
chown tom:wheel  #除了修改属主，将文件的属组改为wheel
chgrp root /tmp/path.out  #专门修改属组的命令，选项可以参考chmod

文件或目录创建时的掩码：umask

文件：666-umask

如果某类用户的权限减得的结果中包含x权限，则将其权限加一

目录：777-umask

umask:查看

umask #:设置umask的值
