Linux的哲学思想：

1.一切皆文件，把几乎所有的资源，包括硬件设备都组织为文件格式

2.有众多的单一目的的小程序组成，一个程序只实现一个功能，而且要做好，组合小程序完成复杂任务

3.尽量避免跟用户交互，目标：实现脚本编程，自动完成某些功能

4.使用纯文本文件保存配置信息，目标：使用文本编辑器即能完成系统配置工作

终端：用户与主机交互，必然用到的设备

物理终端：直接接入本机的显示器和键盘设备：console

虚拟终端：附加在物理终端之上的以软件方式虚拟实现的终端，centos默认6个终端

在shell中的可执行命令有两类：

内建命令：通过某命令形式提供

外部命令：当前系统的某目录下存放的对应可执行二进制文件

区别内部和外部命令：type xxx

运行命令的格式：

COMMAND [OPTIONS… ] [Arguments..]

选项：用于开启或者关闭某些功能

短选项：-c

长选项：–word

参数：命令的作用对象，向命令提供数据

Linux命令

工欲善其事，必先利其器，首先就来看看怎么更快的上手命令吧

文件系统：

文件有两类数据，一种叫做元数据：metadata（存放在索引中），数据：data（文件真正包含的内容）

命令获取帮助

内部命令：help command

外部命令：command --help、man command、info command

hash命令：shell搜寻到的外部命令会缓存至kv(key-value)存储中

history命令：登陆shell时，会读取命令历史文件中记录下的命令：~/.bash_history，登陆进shell后执行的命令只会记录在缓存中，这些命令会在用户退出的时候”追加”到命令历史文件中

-a 追加本次会话新执行的命令历史列表到历史文件中
-d 删除历史中指定的命令，如:history -d 40
-c 清空命令历史

快捷操作

!# 调用历史中的第#条命令
!string 调用历史中最近一个以string开头的命令
!! 上一条命令

程序自身的帮助文档：/usr/share/doc/COMMAND-VERSION,或者查看官方文档

推荐一个网站，需先翻墙再使用
