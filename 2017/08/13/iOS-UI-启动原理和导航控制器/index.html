应用程序启动过程

1.程序启动先执行main.m中的main函数

2.在main函数内部根据第三个参数,创建UIApplication对象

3.根据第四个参数

创建AppDelegate类

然后再去创建AppDelegate对象

将AppDelegate对象赋值给应用程序对象的代理属性，作为代理

4.有了应用程序和应用代理对象，再创建一个窗口（就是AppDelegate 中的window）作为应用程序的主窗口

5.应用程序根据配置的信息去加载默认的storyboard文件里面的控制器

加载自定义控制器

加载自定义控制器

1.将系统提供的viewController.h/.m和main.storyboard删除

2.在应用加载完毕方法内实现加载自定义控制器

自定义控制器，继承自系统的UIViewController

1.先创建一个窗口

2.设定窗口的根控制器为自定义控制器

3.将这个窗口作为主窗口显示并显示

控制器的View是懒加载的

//在第一次使用控制器的view的时候会加载
- (void)loadView{
  [super loadView];
}
//在上面这个方法执行完成之后才会有view，才能对view进行设置

## 导航控制器

UINavigationController的基本使用

- 初始化UINavigationController
- 设置UIWindow的rootViewController为UINavigationController
- 将第一个视图控制器设置为UINavigationController的根视图控制器
- 通过push方法创建子控制器
- 通过pop方法可以返回上一控制器

```objective-c
//导航控制器的基本使用
- (BOOL)application:(UIApplication *)application didFinishLaunchingWithOptions:(NSDictionary *)launchOptions {
    //创建窗口
    self.window = [[UIWindow alloc] initWithFrame:[UIScreen mainScreen].bounds];
    //实例化导航控制器并设置为窗口的根控制器
    UINavigationController *nav = [[UINavigationController alloc] init];
    //设置为根控制器
    self.window.rootViewController = nav;
    // 创建控制器
    PzpViewController *redVc = [[PzpViewController alloc] init];

    Pzp1ViewController *greenVc = [[Pzp1ViewController alloc] init];

    Pzp2ViewController *blueVc = [[Pzp2ViewController alloc] init];
    //让控制器显示出来
    [nav pushViewController:redVc animated:YES];

    [nav pushViewController:blueVc animated:YES];

    [nav pushViewController:greenVc animated:YES];

    [self.window makeKeyAndVisible];

    return YES;
}

导航控制器的常规用法是创建导航控制器的时候就指定一个根控制器，在控制器内制定一些操作的时候，才进行跳转。

- (BOOL)application:(UIApplication *)application didFinishLaunchingWithOptions:(NSDictionary *)launchOptions {
    self.window = [[UIWindow alloc] initWithFrame:[UIScreen mainScreen].bounds];
    PzpViewController *redVc = [[PzpViewController alloc] init];

    UINavigationController *nav = [[UINavigationController alloc]           initWithRootViewController:redVc];

    self.window.rootViewController = nav;

    [self.window makeKeyAndVisible];

    return YES;
}
//在导航控制器中的根控制器中进栈
- (IBAction)gotoGreen:(id)sender {
    Pzp1ViewController *greenViewController = [[Pzp1ViewController alloc] init];
    [self.navigationController pushViewController:greenViewController animated:YES];
    }

//在控制器中可以控制出栈
- (IBAction)back1View{
  [self.navigationController popViewControllerAnimated:YES];
}
- (IBAction)backToRootView{
  [self.navigationController popToRootViewrollrtAnimated:YES];
}
- (IBAction)gotoController    {
  //从控制器栈的数组中去取
  UIViewController *greenViewController self.navigationController.viewController[1];

  [self.navigationController popToViewController:greenViewController animated:YES];
}

导航控制器的工作原理

控制器销毁的顺序：

如果在第三个控制器之中，然后返回根控制器，销毁的顺序是：Two控制器->Three控制器

因为首先断开的是One控制器指向Two控制器的强指针，所以Two控制器首先被回收。

设置导航栏的内容

设置导航栏的文字：

self.navigationItem.title = @"hello";

设置导航栏左侧的按钮

//当左侧和右侧按钮相同的时候，跳转回来的时候左侧的会消失
self.navigationItem.leftBarButtonItem = [[UIBarButtonItem alloc] initWithBarButtonSystemItem:UIBarButtonSystemItemAdd target:self action:@selector(addClick)];
//默认是有拖拽返回功能的，如果左侧添加了按钮就没有了

通过storyboard使用导航控制器

将main.storyboard中的控制器替换为导航控制器，通过控制器连线可以选中根控制器，如果要点击button进行控制器的跳转，也可以通过button连线，选择show。可以用代码改变的在storyboard中修改更方便。

但是要注意的一点是，只有根控制器才有navigation Item,其他的控制器需要手动拖进去。还有一点就是button中没有pop的选项，所以如果要通过button实现pop的话需要和类进行关联实现。

通过storyboard传值

通过控件拖线的方式实现控制器的跳转，不会对按钮的事件进行判断，是无法实现传值的，只能进行基本的跳转操作。

但是通过拖线的方式会调用下面这个方法，可以在这个方法中进行传值。

- (void)prepareForSegue:(UIStoryboardSegue *)segue sender:(id)sender{
    //获取目标控制器
    PController *greenVc = segue.destinationViewController;

    greenVc.navigationItem.title = self.textField.text;
}
