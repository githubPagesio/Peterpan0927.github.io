框架

所谓的框架或库是指，常量，函数，类等东西的一个集合，所谓的Foundation框架就是苹果公司为了帮助开发者开发苹果上的程序所提供的类，方法，函数，常量等数据信息。

核心三大库：字符串(NSString,NSMutableString)，集合(NSArray,NSMutableArray,键值对：NSDictionary,NSMutableDictionary)，文件(NSFileManager)

推荐一个网站苹果所有开源项目的托管网站

学习构造方法和其他方法主要分为五类：增、删、改、查、其他

单例模式

1.单例的特点

1个类的对象无论在何时何地创建，创建出来的都是同一个对象

2.无论如何创建对象，都会调用alloc方法来创建对象

  • alloc方法的内部，只是调用了allocWithZone方法
  • 实际上申请空间，创建对象的事allocWithZone在做

3.要实现单例模式，也就是重写allocWithZone方法

+ (instancetype)allocWithZone:(struct _NSZone *)zone{
    static id instance = nil;
  //只有第一次才会创建对象，后面的每一次都是把第一次创建的对象返回
    if( instance == nil ){
        instance = [super allocWithZone:zone];
    }
    return instance;
}

4.单例模式的规范

如果类是一个单例模式，要求为类提供一个类方法，来返回这个单例对象

类方法的名称必须以shared类名；default类名；

//以Person类为例子
+ (instancetype)sharedPerson;
+ (instancetype)defaultPerson;

+ (instancetype)sharedPerson{
  return [self new];
}
+ (instancetype)defaultPerson{
  return [self new];
}

5.什么时候需要让类变成单例

比如要做一个游戏的面板的尺寸，如果直接写就把程序写死，宏在程序运行的时候无法更改

如果程序需要被整个程序共享，将数据以属性的方式存储在单例对象中，也就是说在A类中改了尺寸，B类访问的时候就是那个修改过的，因为只有一个对象。

2.文件编码

早期的计算机用的是纸带打点，为了解释他们的意思，于是对数据进行了编码，当时由于计算机的内存比较小，所以只用了七个二进制位，也就是128个字符，被称作ASCII码，前三十二个字符是控制字符。

后来IBM把七个二进制位扩展到了一个字节，但是只能在早期的长城机和IBM机上才能看到。

随着计算机的普及，亚洲区域一些非英语使用者的国家，显然ASCII码不够用，于是采用了双字节编码：Unicode ，也就是utf-16，有65535个字符，基本把全世界常用的都包括了进来。

utf其他格式最流行的就是utf-8格式，所有的汉字是三个字节，英文是一个字节，现在的网站大部分都是utf-8格式。

NSDictionary

NSArray和NSMutableArray都可以存储数据，但是缺点是数组元素的下标不固定，无法通过下标来确定唯一数组中的元素，所以我们希望有一种存储方式，存储到数组当中，可以快速唯一的确定数组的元素。

存储数据的时候，要为存储的数据取一个别名，用来确定这个别名对应的数据。

这种存储数据的方式就叫做键值对的存储方式

创建字典数组

//如果只是简单初始化，此时字典数组中没有任何元素，没有意义
NSDictionary *dict = [NSDictionary new];
//一般创建方式
NSDictionary *dict = [NSDictionary dictionaryWithObjectsAndKeys:@"jack",@"name",@"北京市",@"address", nil];
//简要创建方式
NSDictionary *dict = @{@"name":@"jack",@"age":@"18"};

存储数据的原理

1.以键值对的形式存储数据

2.字典数组一旦创建，其中的元素不能动态的新增和删除

3.键：只能是遵守了NSCoping协议的对象,而NSString就是遵守了这个协议

值：只能是OC对象

4.当往字典数组中存入一个键值对的时候，并不是按照顺序挨个存储，而是根据键和数组的长度做一个哈希算法算出一个下标，再将这个键值对存入这个下标对应的空间。

5.取值的时候，也可以通过键做一个哈希算法，算出下标，然后直接找到对应的数据取出即可

字典数组的使用

//取出存储在字典数组中的数据

字符串和日期转换

可以得到当前时间，创建一个NSDate对象即可，将这个对象输出就是当前系统的格林威治时间，如果要格式化输出日期，就要采取下面方式：

NSDate *date = [NSDate new];
NSDateFormatter *formatter = [NSDateFormatter new];
//代表显示年月日十分秒
formatter.dateFormat = @"yyyy-MM-dd HH:mm:ss";
NSString *str = [formatter stringFromDate:date];
//这样输出的就是当前时区的格式化时间,可以在时间之间随意的添加自定义的效果,e.g->yyyy年MM月dd日
NSLog(@"%@", str);

将字符串转换为时间

NSString *str = @"2016-09-27 12:12:12";
NSDateFormatter *formatter = [NSDateFormatter new];
formatter.dateFormat = @"yyyy-MM-dd HH:mm:ss";
NSDate *date = [formatter dateFromString:str];

日期计算

在当前基础上，新增一个指定的时间，得到一个新的时间

//获取的默认是零时区的时间，到东八区的时间就是加八个小时
NSDate *date = [NSDate dateWithTimeIntervalSinceNow:5000];
//求两个时间之差
NSDate *date1 = [NSDate new];
NSDate *date2 = [NSDate new];
double res = [date1 timeIntervalSinceDate:date2];

得到日期对象的各个部分

NSDate *date = [NSDate date];
//创建一个日历对象，可以从日历对象中取到日期的各个部分
NSCalendar *calendar = [NSCalendar currentCalendar];
NSDateComponents *com = [calendar components:NSCalendarUnitYear fromDate:date];
//可以直接使用成员运算符取出日期对象的各个部分
NSLog(@"%lu", com.year);
